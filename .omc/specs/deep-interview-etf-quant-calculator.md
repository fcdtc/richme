## Goal
构建一个**ETF量化交易计算器**——用户输入持仓信息和风险偏好，系统基于技术指标分析实时数据，输出交易建议（买入/卖出/持有）及详细分析过程。每次计算独立，不涉及实际交易执行。

## Product Definition

### 产品形态
**量化交易计算器**（非持续监控系统）
- 用户主动输入 → 系统分析 → 输出建议
- 每次计算独立，无状态持久化需求
- 本地运行，浏览器访问

### 核心功能
1. **交易信号生成**
2. **数据看板**（Web界面）

## User Input Parameters
用户每次使用时需要输入：

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| ETF代码 | string | 要分析的ETF基金代码 | 512400 |
| 总资金 | float | 用户总可用资金 | 100000 |
| 持仓金额 | float | 当前该ETF持仓市值 | 20000 |
| 当前收益率 | float | 当前持仓收益率（%） | 5.5 |
| 风险偏好 | enum | 保守/中性/激进 | 中性 |

**注意**：用户可能分批买入，无法精确知道份额和成本价，只能看到持仓金额和收益率。

## Trading Strategies
系统需要实现以下策略，并根据风险偏好调整权重：

### 1. 均线交叉策略
- MA5/MA10/MA30交叉信号
- 价格与均线的位置关系
- 均线多头/空头排列

### 2. 动量指标策略
- **MACD**：金叉死叉、柱状图变化
- **RSI**：超买(>70)/超卖(<30)判断
- 可扩展其他指标（KDJ、CCI等）

### 3. 突破/通道策略
- 布林带通道突破
- 支撑/压力位突破
- 箱体震荡判断

### 4. 量价配合策略
- 放量上涨/下跌
- 缩量整理
- 量价背离

### 风险偏好调整
| 风险偏好 | 信号阈值 | 止损建议 | 仓位建议 |
|----------|----------|----------|----------|
| 保守 | 更严格 | -3%~-5% | 建议低仓位 |
| 中性 | 标准 | -5%~-8% | 建议中等仓位 |
| 激进 | 更宽松 | -8%~-12% | 可高仓位 |

## System Output
输出包含两部分：

### 1. 交易建议
| 信号 | 条件 |
|------|------|
| **强烈买入** | 多数指标看多，风险偏好匹配 |
| **买入** | 多数指标看多 |
| **持有** | 指标中性或信号矛盾 |
| **卖出** | 多数指标看空 |
| **强烈卖出** | 多数指标看空，风险偏好匹配 |

### 2. 分析过程
- 各项技术指标当前值
- 指标解读（多头/空头/中性）
- 综合评分明细
- 风险提示
- 建议操作说明

- [ ] 系统能计算并展示均线(MA5/10/30)、MACD、RSI等技术指标
- [ ] 系统能基于策略组合生成交易信号
- [ ] 输出包含明确的交易建议（买入/卖出/持有）
- [ ] 输出包含详细的指标分析过程
- [ ] 风险偏好能影响信号生成的阈值
- [ ] Web界面美观，数据可视化清晰
- [ ] 本地启动后可通过浏览器访问

## Assumptions Exposed & Resolved
| Assumption | Challenge | Resolution |
|------------|-----------|------------|
| 需要持续监控 | Contrarian: 如果只是每次独立计算呢？ | 确认为"计算器"模式，每次独立 |
| 需要模拟盘 | Contrarian: 用户真的需要模拟吗？ | 确认只需信号提醒，无需模拟 |
| 用户知道成本价 | Simplifier: 用户可能分批买入 | 改为输入持仓金额+收益率 |
| 需要止盈止损参数 | Simplifier: 能否简化输入？ | 用风险偏好替代具体止盈止损 |
